# L 桥梁工具使用说明

## L 工具是什么
L 桥梁工具是一个智能自动化工具，用于在编程工具和 ChatGPT 之间建立双向桥梁，帮助您更高效地使用 AI 助手。

## 第一次使用步骤
1. **双击 start_L.bat** 启动工具（首次运行会自动安装依赖）
2. **选择配置**：在「项目配置区」选择「左侧项目」或「右侧项目」
3. **采集模板**：按照「下一步提示」的指引采集所需模板
4. **输入提示词**：在「指令输入」框中输入您的需求
5. **一键发送**：点击「一键发送到编程工具」按钮
6. **回传结果**：使用「一键回传到 ChatGPT」将编程结果发送给 GPT

## 左侧项目/右侧项目配置说明
- **左侧项目**：适用于左边的编程工具窗口
- **右侧项目**：适用于右边的编程工具窗口
- 每个配置都有独立的模板、队列和安全设置
- 可以通过「新建配置」创建更多项目配置

## 模板采集顺序
1. **编程工具输入框**：输入代码提示词的地方
2. **编程工具复制按钮**：复制代码结果的按钮
3. **ChatGPT 输入框**：粘贴结果到 GPT 的地方
4. **发送按钮**（可选）：如果不采集，将使用 Enter 键发送
5. **其他模板**（可选）：删除/确认按钮、沙箱按钮等

## 安全模式说明
- **安全模式**：只粘贴文本，不点击发送按钮
- **真实执行模式**：自动粘贴并发送
- 首次切换到真实执行模式会有确认提示
- 回传区也有独立的安全模式设置

## 一键发送到编程工具
- 自动包裹提示词标记（<<PROMPT>> 和 <<END>>）
- 自动加入队列并执行
- 安全模式下只粘贴不发送
- 会检查输入框模板是否已采集

## 一键回传到 ChatGPT
- 自动复制编程工具结果
- 自动粘贴到 ChatGPT 输入框
- 回传安全模式下只粘贴不发送
- 会检查必要模板是否已采集

## 一键安全演练
- 测试所有关键模板的识别
- 执行安全测试粘贴
- 只移动鼠标，不执行危险操作
- 帮助您验证模板质量

## 关键词识别兜底
当模板识别失败时，可以使用关键词识别作为兜底方案：
- **模板识别**：依赖截图和图像匹配，适合固定位置的按钮
- **关键词识别**：扫描窗口中的按钮文本，适合位置不固定的情况
- L 工具会优先尝试模板方式，失败后自动使用关键词方式

### 关键词设置
在「关键词识别设置区」可以自定义关键词：
- **删除/确认关键词**：用于点击弹窗中的确认按钮，默认：删除,确认,Yes,OK
- **后台沙箱关键词**：用于点击后台运行按钮，默认：后台,沙箱,后台运行,Run in background
- 多个关键词用逗号分隔
- 修改后记得点击「保存关键词设置」

### 常见问题

### a. 找不到输入框怎么办
- 确保编程工具窗口在最前面
- 重新采集输入框模板
- 调整选择区域，确保包含输入框的特征部分
- 检查模板文件是否保存成功

### b. 粘贴中文失败怎么办
- 确保已安装 pyperclip 库
- 如果仍失败，可能是 pyautogui 限制，尝试使用 Ctrl+V 手动粘贴
- 检查系统剪贴板是否正常

### c. 模板识别分数低怎么办
- 重新采集模板，选择更清晰的区域
- 确保模板区域有明显特征
- 避免选择包含动态内容的区域
- 调整窗口大小后需要重新采集

### d. 误点了真实执行怎么办
- 立即点击「停止执行」按钮
- 使用 Alt+Tab 切换到其他窗口
- 如果已经发送，尝试使用编程工具的撤销功能

### e. 如何重新采集模板
- 点击「刷新模板状态」按钮
- 点击对应的「采集」按钮重新选择区域
- 新采集的模板会自动覆盖旧模板
- 采集后会自动更新模板状态

### f. 弹窗按钮文字不是"删除"怎么办
- 可以在「关键词识别设置区」添加该按钮的文字作为关键词
- 例如：弹窗按钮是"确定"，就在删除/确认关键词中添加"确定"
- 修改后点击「保存关键词设置」

### g. 关键词识别找不到按钮怎么办
- 确保 pywinauto 已安装（运行 start_L.bat 会自动安装）
- 确保目标窗口可见，没有被其他窗口遮挡
- 检查关键词是否与按钮文字匹配（大小写不敏感）
- 可以使用系统自检来验证 pywinauto 是否正常工作

## 版本备份
- **创建版本备份**：点击顶部「创建版本备份」按钮，可以将当前程序备份到 backups 文件夹
- 建议每次大改前点击创建版本备份，如果后续改坏，可以从 backups 文件夹恢复
- 备份文件格式：l_bridge_tool_gui_年月日_时分秒.py
- 备份目录会在点击按钮后自动打开

## 其他功能
- **窗口置顶**：保持工具窗口在最前面
- **清空队列**：清除所有待执行的指令
- **打开模板目录**：查看当前配置的模板文件
- **重置当前流程**：停止任务并清空队列

祝使用愉快！

## 已知提示
如果启动时看到：
```
QWindowsContext: OleInitialize() failed
```
通常是 PyQt5/Windows 环境警告，不影响 L 工具正常使用。只要 GUI 正常打开即可。
